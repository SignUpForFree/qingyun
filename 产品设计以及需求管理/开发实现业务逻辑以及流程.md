## 1. 首次进入 / 登录流程

### 1.1 用户首次进入站点

**入口**：浏览器输入 `http://192.144.226.27:3000` 或微信内点链接。

| 步骤 | 用户看到 | 触发的代码 | 状态 |
|---|---|---|---|
| 1.1.1 GET / | RSC `app/page.tsx` 渲染 → `requireUserId()` 抛 `UnauthenticatedError` | `lib/auth/session.ts` | [已实现] |
| 1.1.2 渲染 `<LoginGate />` | 一个白卡 "请先登录"，mount 自动弹 LoginSheet | `components/auth/LoginGate.tsx` | [已实现] |
| 1.1.3 LoginSheet 从底部弹出 | 标题 "欢 迎 回 来" + PhoneLoginForm | `components/auth/LoginSheet.tsx` | [已实现] |

### 1.2 手机号登录（PhoneLoginForm）

| 按钮 / 输入 | 行为 | 状态 |
|---|---|---|
| **手机号输入框**（11 位） | onChange 过滤非数字；validate `/^1[3-9]\d{9}$/` | [已实现] |
| **发送验证码** 按钮 | `POST /api/auth/phone/send-otp` body `{phone:"+86xxx"}`；成功 toast + 60s 倒计时；429 → 提示冷却时间。**dev 模式 mock=true，控制台打印验证码** | [已实现·有边界] |
| **验证码输入框**（6 位） | onChange 过滤非数字；`/^\d{6}$/` | [已实现] |
| **登录** 按钮 | `POST /api/auth/phone/verify` `{phone, code}` → 后端：① 验码（错 3 次锁定）② 已绑过 → 复用 user_id；没绑过 → 建 user + insert phone_bind + 占位 profile ③ setSessionCookie 返回 `{userId, isNew}` | [已实现·有边界] |
| 登录成功（isNew=true） | onSuccess 关 Sheet → `router.replace("/onboarding")` | [已实现] |
| 登录成功（isNew=false） | onSuccess 关 Sheet → `router.refresh()`（保持当前页） | [已实现] |
| **错误处理** | `expired/wrong/too_many_attempts/rate_limited` 各自 toast 文案 | [已实现] |

**已知边界**：

- **OTP store 是进程内 Map**（`lib/auth/phone-otp.ts`）。多容器 / 重启后 OTP 会丢。M5 改 SQLite 或 Redis（见 ARCHITECTURE §4.1）。
- **短信渠道未接**。当前是 dev 模式 console.info 打码。M5 接 SMS gateway（阿里云 / 腾讯云 SMS 任选）。
- **国际手机号未支持**。validate 锁死 +86。

### 1.3 微信 OAuth 登录（V1.0 服务号路径）

| 按钮 / 路由 | 行为 | 状态 |
|---|---|---|
| `GET /api/auth/wechat` | 生成 nonce + 签名 state（HMAC，5min TTL）→ 302 微信授权页 | [已实现] |
| `GET /api/auth/wechat/callback?code&state` | ① verifyState ② code → access_token ③ fetchUserinfo ④ upsert wechat_bind（openid 唯一）⑤ 首次建 users + 占位 profile ⑥ setSessionCookie ⑦ 302 /onboarding（新）/ /（老） | [已实现·有边界] |
| 错误码 40029（code 重用） | 302 重启 OAuth `/api/auth/wechat`（防御浏览器后退/分享导致 code 再用） | [已实现] |

**已知边界**：

- **小程序登录路由 `/api/auth/wechat-mini` 未实现**。当前只支持服务号 H5 OAuth。**Beta 上线必做**（见 ARCHITECTURE §4.1）—— 直接复用 `lib/wechat/oauth.ts` 的 `bindOrCreateUser(openid, unionid)` helper。
- 小程序版要复用 unionid 优先合并策略，让用户跨小程序 / 公众号 / H5 都能找回历史（参 ARCHITECTURE §3.2-L "持久化与多端同步"）。

### 1.4 登出

| 按钮 / 路由 | 行为 | 状态 |
|---|---|---|
| `/me/settings` 的 **退出登录** 按钮 | `POST /api/auth/logout` → 清 cookie + 302 / | [已实现] |
| 替代：`POST /api/logout`（兼容旧路径） | 同上 | [已实现] |

## 19. 怎么用本文档

1. **遇到 bug 时**：ctrl+F 搜按钮名字 / API 路径 → 看「实现位置」，秒定位代码。
2. **加新功能时**：先在对应章节加一行 [未实现]，明确按钮位置 + 触发链路；再写代码改 [已实现]。
3. **review 进度时**：grep `[占位]` / `[未实现]` 总数 → 知道还差多少完成度。
4. **做 P0/P1 派工时**：直接照 §18 缺口清单 → 一次只攻一条。
5. **每改完一条**：把状态从 [占位] / [未实现] 改成 [已实现] 提一笔 commit，本文档跟代码一起演进。
